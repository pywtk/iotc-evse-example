TO library migration guide from 4.x.x to 5.x.x¶

The following changes are to be taken into account to update from 4.x.x to 5.x.x.

Old APIs are now deprecated, but still defined and mapped on new APIs by including TO.h.

Internal architecture changes¶

The source tree structure has changed:

  • Core API based on Trusted Objects Secure Element has been moved from

src/to/ to src

Generic APIs renames¶

Generic TO_…() functions have been renamed to TOSE_…().

New TOSE_…() APIs now take a context parameter, given by driver function TODRV_SSE_get_ctx().
