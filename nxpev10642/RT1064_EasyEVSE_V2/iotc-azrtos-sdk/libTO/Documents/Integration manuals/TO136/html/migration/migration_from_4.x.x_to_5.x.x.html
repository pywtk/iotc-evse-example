TO library migration guide from 4.x.x to 5.x.x¶

The following changes are to be taken into account to update from 4.x.x to 5.x.x.

Old APIs are now deprecated, but still defined and mapped on new APIs by including TO.h.

Internal architecture changes¶

The source tree structure has changed:

  • Core API based on Trusted Objects Secure Element has been moved from

src/to/ to src

  • Wrappers directory has been moved from src/to/wrapper/ to src/wrapper/.

Generic APIs renames¶

Generic TO_…() functions have been renamed to TOSE_…().

New TOSE_…() APIs now take a context parameter, given by driver function TODRV_HSE_get_ctx().

Hardware Secure Element renames¶

Hardware Secure Element specific functions have need renamed to TODRV_HSE_…:

  • TO_config() renamed to TODRV_HSE_trp_config()

  • TO_last_command_duration() renamed to TODRV_HSE_trp_last_command_duration()

  • TO_read() renamed to TODRV_HSE_trp_read()

  • TO_write() renamed to TODRV_HSE_trp_write()

  • TO_seclink_…() renamed to TODRV_HSE_seclink_...()

  • TO_prepare_command_data() renamed to TODRV_HSE_prepare_command_data()

  • TO_prepare_command_data_byte() renamed to TODRV_HSE_prepare_command_data_byte()

  • TO_set_command_data() renamed to TODRV_HSE_set_command_data()

  • TO_send_command() renamed to TODRV_HSE_send_command()

  • TO_set_lib_hook_pre_command() renamed to TODRV_HSE_set_lib_hook_pre_command()

  • TO_set_lib_hook_post_write() renamed to TODRV_HSE_set_lib_hook_post_write()

  • TO_set_lib_hook_post_command() renamed to TODRV_HSE_set_lib_hook_post_command()

Hardware Secure Element specific files have need renamed to TODRV_HSE_…:

  • TO_i2c_wrapper.h renamed to TODRV_HSE_i2c_wrapper.h

  • TO_hooks.h renamed to TODRV_HSE_hooks.h
